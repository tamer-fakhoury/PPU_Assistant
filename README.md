# مساعد جامعة بوليتكنك فلسطين — PPU Assistant

> An intent-routed, retrieval-augmented (RAG) chatbot for Palestine Polytechnic University.  
> Answers are *never* generated by an LLM — only accurate URLs and structured page excerpts are returned, eliminating hallucination entirely.

## Tech Stack

| Layer | Technology | Purpose |
|-------|-----------|---------|
| **Server** | Python 3.12 + FastAPI + Uvicorn | Async HTTP server |
| **Intent Classifier** | Groq API (llama-3.1-8b-instant) | Primary intent resolution, ≤70 tokens |
| **Embedding Model** | `intfloat/multilingual-e5-small` | Arabic-optimized multilingual embeddings (384-dim) |
| **Vector Search** | FAISS (IndexFlatIP) | Approximate nearest-neighbor inner-product search |
| **Corpus Builder** | httpx + BeautifulSoup4 + Camel Tools | Web scraping + Arabic normalization |
| **Frontend** | Vanilla HTML/CSS/JS (RTL) | Chat interface with clickable links |
| **Config** | python-dotenv, YAML | API keys, intent definitions |

## Query Pipeline (Detailed)

### Step 1 — Intent Resolution (`engine/router.py:resolve_intent_via_llm`)

```
                    ┌──────────────┐
                    │  User Query  │
                    └──────┬───────┘
                           │
                    ┌──────▼───────┐
                    │  Greeting?   │──yes──► greeting
                    │  (regex)     │
                    └──────┬───────┘
                           │ no
                    ┌──────▼───────┐
                    │  LLM Classify│──success──► parse_llm_output()
                    │  (Groq)      │               │
                    └──────┬───────┘               │
                           │ fail                  │ safety net:
                    ┌──────▼───────┐               │ template→rag for
                    │  Embedding   │               │ RAG intents
                    │  Router      │               │
                    │  (E5+FAISS)  │               │
                    └──────────────┘               │
                           │                       ▼
                           └───────── IntentResult(action, intent_name, lang)
```

**Modes**:
- `greeting` — Regex-detected (مرحبا, hello, شكرا, etc.), zero dependencies
- `template:X` — Hardcoded response from intents.yaml (contact, location, about, navigation)
- `rag:X` — Search corpus + return URL + excerpts (colleges, admission, tuition, academic_calendar, jobs, student_services)
- `llm` — Fallback when intent is unclear; treated as `rag` with generic template

### Step 2 — Search Query Resolution (`main.py:_resolve_search_query`)

```
LLM generate_search_query(query, intent) ──success──► search query
        │
        │ fail
        ▼
get_fallback_search_query(intent) ──► first exemplar from intents.yaml
        │
        │ fail
        ▼
    raw user query
```

### Step 3 — Dense Retrieval (`engine/rag.py:search`)

```
query ──► normalize_arabic()
        ──► "query: " + normalized ──► E5-small encode (384-dim)
        ──► FAISS IndexFlatIP.search(top_k=25)
        ──► 25 chunks with {text, url, title, score}
```

### Step 4 — Intent-Prioritized Chunk Selection (`main.py:_pick_intent_chunk`)

```
chunks (top 25 FAISS) + intent_name
        │
        ▼
search each chunk's URL for intent keywords
(admission→"admission", colleges→"colleges"|"deanships", etc.)
        │
    ┌───┴───┐
    │found? │
    └───┬───┘
   yes │       no
       ▼        ▼
 intent-chunk   chunks[0] (best FAISS score)
```

This step overrides pure embedding ranking. An admission page ranked #19 by FAISS is preferred over an academic calendar page ranked #1.

### Step 5 — Response Building (`main.py:_build_rag_response`)

```
best chunk + chunks + intent_name + lang
        │
        ▼
┌───────────────────────────────────────────┐
│ 1. Template from intents.yaml             │
│    "لقد وجدت معلومات حول ... {url}"       │
│                                           │
│ 2. Substitute {url} → <a class="link">    │
│                                           │
│ 3. Convert \n → <br>                      │
│                                           │
│ 4. Add excerpts (2 unique chunks,         │
│    intent-chunk first)                    │
│    "مقتطفات من الصفحة:"                   │
│    • cleaned excerpt 1                    │
│    • cleaned excerpt 2                    │
└───────────────────────────────────────────┘
```

## Intents System

Defined in `engine/intents.yaml`, the system uses **10 intents** in two categories:

### Template Intents (hardcoded, zero retrieval)

| Intent | Examples | Threshold |
|--------|----------|-----------|
| `contact` | رقم الهاتف, how to contact PPU | 0.65 |
| `location` | أين تقع الجامعة, university address | 0.65 |
| `about` | نبذة عن الجامعة, about PPU | 0.70 |
| `navigation` | بوابة الطالب, student portal | 0.65 |

### RAG Intents (retrieve + return URL + excerpts)

| Intent | Source URL | Keywords | Threshold | Exemplars (ar+en) |
|--------|-----------|----------|-----------|-------------------|
| `colleges` | `/ar/Colleges-Deanships` | colleges, deanships | 0.70 | 23 |
| `admission` | `/ar/admission` | admission | 0.70 | 19 |
| `tuition` | `/ar/admission` | admission, fees | 0.70 | 25 |
| `academic_calendar` | `/ar/academic-calendar` | academic, calendar, calender | 0.70 | 14 |
| `jobs` | `/ar/jobs` | jobs | 0.70 | 11 |
| `student_services` | `/ar/student-services` | student, services | 0.65 | 18 |

Each RAG intent has a `response_ar`/`response_en` template with `{url}` placeholder, e.g.:
```yaml
response_ar: "لقد وجدت معلومات حول شروط القبول والتسجيل في جامعة بوليتكنك فلسطين.
              للاطلاع على تفاصيل القبول والتسجيل، يرجى زيارة الرابط التالي:\n\n{url}"
```

## Safety Systems

### 1. Hallucination Prevention
- LLM is used ONLY for intent classification (≤70 tokens) and search query generation (≤30 tokens)
- **No LLM-generated answers** — responses are templates or URLs from the corpus
- The `ask()` function was explicitly removed from `engine/llm.py`

### 2. Intent Safety Net (`engine/router.py:68-84`)
When the LLM misclassifies a RAG intent as `template:X`, the system auto-corrects:
```python
_RAG_INTENTS = {"colleges", "admission", "tuition", "academic_calendar", "jobs", "student_services"}
if result.action == "template" and result.intent_name in _RAG_INTENTS:
    result.action = "rag"  # correction
```

### 3. Embedding Fallback Router (`engine/router.py:87-122`)
When the LLM is unavailable (API key missing, network error, quota exceeded):
- Pre-computed exemplar embeddings (`intents_embedded.pkl`) are cosine-matched against the query
- The best match above threshold determines intent
- Falls back to `llm` action (generic RAG) if nothing matches

### 4. Intent-Prioritized Chunk Picking (`main.py:_pick_intent_chunk`)
Overrides FAISS ranking when intent is known. Admission query → admission URL even if FAISS ranks it below other pages.

### 5. Relevance Threshold (`RELEVANCE_THRESHOLD = 0.15`)
If even the best chunk scores below 0.15, the system returns the help template instead of a low-relevance URL.

## Response Format

All endpoints return JSON via `PPUResponse.to_dict()`:

```json
{
  "text": "لقد وجدت معلومات حول شروط القبول والتسجيل...\n<a class=\"link\" href=\"...\">...</a>\n\n<strong>مقتطفات من الصفحة:</strong>\n• excerpt 1\n• excerpt 2",
  "sources": ["https://www.ppu.edu/p/ar/admission", "https://..."],
  "method": "rag"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `text` | str | HTML-safe response body (rendered via innerHTML) |
| `sources` | list[str] | Deduplicated page URLs for "source badges" |
| `method` | str | `greeting` | `template` | `rag` | `help` |

## Endpoints

### `GET /`
Serves the RTL chat interface (`static/index.html`).

### `GET /health`
```json
{"status": "ok"}
```

### `GET /query?q=<query>`
Primary endpoint. Returns JSON response synchronously.

### `GET /query/stream?q=<query>`
Server-sent events (SSE) endpoint. Streams status updates, then the final response:
```
data: {"type": "status", "text": "جاري البحث..."}

data: {"text": "...", "sources": [...], "method": "rag"}
```

## Corpus Building

### `preload/build_index.py`
- Scrapes 30 seed URLs from `ppu.edu` via httpx + BeautifulSoup
- Extracts `<main>` / `<article>` content, strips nav/footer/header/scripts
- Chunks at ~300 words per chunk, prepending `رابط: {url} | صفحة: {title} | `
- Embeds with `"passage: " + chunk_text` via E5-small
- Builds FAISS IndexFlatIP (384-dim, inner product)
- Output: `data/ppu_chunks.json` + `data/faiss_index.bin`

### `preload/build_intents.py`
- Reads all 10 intents from `engine/intents.yaml`
- Collects all exemplars (Arabic + English, ~164 total)
- Embeds with `"query: " + normalize_arabic(exemplar)` via E5-small
- Saves per-intent embedding arrays to `data/intents_embedded.pkl`

## Setup & Development

### Prerequisites
- Python 3.10+
- ~5GB free disk (model caching)

### Installation
```bash
git clone <repo>
cd ppu_assistant
python -m venv .venv
.venv\Scripts\activate     # Windows
pip install -r requirements.txt

# Set your API key
copy .env.example .env     # Edit .env with your GROQ_API_KEY

# (Optional) Rebuild corpora
python preload/build_intents.py
python preload/build_index.py
```

### Run
```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```

Open `http://localhost:8000` in a browser.

## Known Limitations

1. **Small corpus** (33 chunks, 29 pages) — limited coverage of PPU subpages
2. **Tuition 404** — `ppu.edu/p/ar/fees` returns 404; tuition queries redirect to admission page
3. **Slideshow noise** — 6 chunks from `ppu.edu/p#slideshow-N` contain carousel content, not page-specific info
4. **Embedding model on CPU** — E5-small runs on CPU; batch encoding adds ~8s during index build
5. **No answer generation** — by design, but users expecting conversational answers may find URL-only responses terse
6. **Academic calendar URL typo** — PPU site uses `academic-calender` (missing "a") in the actual path

## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| No LLM answer generation | Zero hallucination risk; answers are always facts (URLs) from the corpus |
| Intent-prioritized chunk picking | FAISS alone can't be trusted for Arabic; admission must return admission page even at rank #19 |
| top_k=25 instead of 3 | Intent-matching chunks often rank #10–25; top_k=3 would miss them |
| E5-small + query/passage prefixes | Best Arabic retrieval quality for the available disk/memory budget |
| Fallback search query from first exemplar | Works with zero API calls; no external service dependency |
| Embedding router as LLM fallback | Graceful degradation when Groq is unavailable |
